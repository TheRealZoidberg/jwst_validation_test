Introduction#

This is the validation notebook that displays the unit tests for the Outlier Detection step in calwebb_image3, calwebb_spec3, calwebb_coron3, and calwebb_tso3. This notebook runs and displays the unit tests that are performed as a part of the normal software continuous integration process. For more information on the pipeline visit the links below.

  • Pipeline description: https://jwst-pipeline.readthedocs.io/en/latest/jwst/outlier_detection/index.html

  • Pipeline code: https://github.com/spacetelescope/jwst/tree/master/jwst/

JWST Unit Tests#

JWST unit tests are located in the tests folder for each pipeline step within the GitHub repository, e.g., jwst/outlier_detection/tests.

  • Unit test README: https://github.com/spacetelescope/jwst#unit-tests

Test Description#

Unit testing is a software testing method by which individual units of source code are tested to determine whether they are working sufficiently well. Unit tests do not require a separate data file; the test creates the necessary test data and parameters as a part of the test code.

Top of Page

Data Description#

Data used for unit tests is created on the fly within the test itself, and is typically an array in the expected format of JWST data with added metadata needed to run through the pipeline.
